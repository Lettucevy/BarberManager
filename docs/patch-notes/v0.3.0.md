# Patch notes - v0.3.0

**Data de lancamento:** 28/06/2026
**Tag Git:** `v0.3.0`
**Branch principal:** `develop`

## Resumo

Versao com foco em base solida (FASE 1) e autoatendimento do cliente
(FASE 2). Clientes agora podem criar conta, login, ver historico
proprio, cancelar e reagendar sem precisar ligar.

## Mudancas

### FASE 1 ŌĆö Fundacao

#### Banco de dados
- `server/db/init.sql`: Coluna `observacao TEXT` adicionada em
  `agendamentos`.
- `server/seed.js`: Migration `senha_hash` para `clientes`.
- Migration automatica via `ensureColumns()` em `seed.js`.

#### Observacao nos agendamentos
- `server/src/routes/agenda.js`: POST e PUT aceitam `observacao`.
- `client/src/pages/ClienteAgendamento.tsx`: Campo observacao agora
  eh enviado ao backend.

#### Bloqueio de duplicidade de cliente
- `server/src/routes/clientes.js`: POST agora faz "find-or-create"
  por telefone. Se cliente ja existe, retorna ele (e atualiza
  nome/email se mudaram).

#### Paginacao no historico
- `server/src/routes/historico.js`: Aceita `?page=&limit=`. Retorna
  `{ data, total, page, limit, totalPages }`.
- `client/src/pages/Historico.tsx`: Consome formato paginado, busca
  por texto, botoes Anterior/Proxima, contagem total.

### FASE 2 ŌĆö Experiencia do Cliente (Vitrine)

#### Conta do cliente (autoatendimento)
- `server/db/init.sql`: Coluna `senha_hash TEXT` em `clientes`.
- `server/src/routes/authCliente.js`: Novo modulo com rotas:
  - `POST /api/auth/cliente/register` ŌĆö cria conta com nome, telefone, senha.
  - `POST /api/auth/cliente/login` ŌĆö login por telefone + senha.
  - `POST /api/auth/cliente/logout` ŌĆö encerra sessao.
  - `GET /api/auth/cliente/me` ŌĆö retorna dados do cliente logado.
- `server/src/index.js`: Monta rota `/api/auth/cliente`.

#### Pagina Minha Conta
- `client/src/pages/MinhaConta.tsx`: Tela de login/registro do
  cliente com toggle entre modos.
- `client/src/pages/MeusAgendamentos.tsx`: Lista de agendamentos do
  cliente com:
  - Visualizacao de agendamentos futuros e historico.
  - Botao **Cancelar** com confirmacao.
  - Botao **Reagendar** com seletor de nova data/hora e validacao
    de disponibilidade.
  - Botao **+ Calendario** (.ics) para exportar para Google/Apple
    Calendar.
- `client/src/App.tsx`: Rotas `/minha-conta` e
  `/minha-conta/agendamentos` adicionadas sob `VisagioLayout`.
- `client/src/layouts/VisagioLayout.tsx`: Link "Minha conta" no menu
  da vitrine.

#### Fluxo de agendamento inteligente (logado vs convidado)
- `client/src/pages/ClienteAgendamento.tsx`: Se o cliente estiver logado:
  - Step "4. Seus dados" some do indicador de passos e do formulario.
  - Secao "Confirmacao" exibe nome + telefone da conta + campo
    observacao.
  - Ao confirmar, usa `cliente_id` da sessao (sem `POST /api/clientes`).
- Se nao estiver logado: fluxo original mantido (step 4 com dados).
- `client/src/pages/ClienteAgendamento.tsx`: Summary tem link
  "Faca login" para convidados.

#### Status de login no menu
- `client/src/layouts/VisagioLayout.tsx`: Verifica sessao via
  `/api/auth/cliente/me`. Exibe **nome do cliente logado** (com
  link para agendamentos) ou "Minha conta" (para login).

#### CSS
- `client/src/App.css`: `.vg-topbar__link--user` ŌĆö destaca o nome
  do usuario logado no menu.
- `client/src/App.css`: Estilos para:
  - `.vg-account` ŌĆö layout do card de login/agendamentos.
  - `.vg-appointments`, `.vg-appointment` ŌĆö lista de agendamentos.
  - `.vg-badge--confirmed/cancelado/concluido` ŌĆö status coloridos.
  - `.vg-reschedule` ŌĆö formulario de reagendamento inline.
  - `.bm-search-form`, `.bm-pagination`, `.bm-empty` ŌĆö historico.

### Outras correcoes
- `client/src/App.css`: Adicionados estilos faltantes para
  `.bm-search-form`, `.bm-pagination`, `.bm-empty`.
- `client/src/layouts/VisagioLayout.tsx`: Links do menu corrigidos
  (`/#cortes`, `/#equipe`, `/#agendar`) para funcionar de qualquer rota.

### FASE 4 ŌĆö Infraestrutura

#### PWA instavel (4.2)
- `client/package.json`: Adicionada dependencia `vite-plugin-pwa`.
- `client/vite.config.ts`: Configurado `VitePWA` com:
  - Manifest completo (nome, icone SVG, theme_color, display standalone).
  - Service worker com auto-update e cache offline via Workbox.
  - Cache runtime para chamadas a API (NetworkFirst).
- `client/public/icons/icon.svg`: Icone da tesoura para o PWA.
- Ao acessar pelo Chrome/Safari, o site pode ser instalado como app.

#### Backup automatico do SQLite (4.3)
- `server/backup.js`: Modulo de backup com:
  - `backup()` ŌĆö copia o .db para `server/backups/` com timestamp.
  - `listBackups()` ŌĆö lista backups existentes.
  - `cleanOld(keep=30)` ŌĆö remove backups antigos, mantendo os 30 mais recentes.
- `server/src/routes/backup.js`: Rotas:
  - `POST /api/backup/manual` ŌĆö dispara backup manual.
  - `GET /api/backup/list` ŌĆö lista backups disponiveis.
- `server/src/index.js`: Backup automatico a cada 24h via `setInterval`,
  executa tambem na inicializacao do servidor.

#### Logs estruturados com pino (4.4)
- `server/src/utils/db.js`: Substituido `console.log/error` por `pino`.
- `server/src/index.js`: `req.log` injetado em todas as requisicoes.
- Todos os modulos de rota: `console.error` substituido por `req.log.error`.
- `server/backup.js`, `server/seed.js`: Usam `pino` diretamente.
